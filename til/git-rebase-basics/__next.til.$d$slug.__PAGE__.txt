<h2>Apa itu Git Rebase?</h2></p>
<p>Git rebase adalah perintah yang digunakan untuk mengubah basis commit dari sebuah branch. Ini sangat berguna untuk merapikan commit history sebelum merge ke branch utama.</p>
<h2>Kapan Menggunakan Rebase?</h2>
<p>Gunakan rebase ketika:
<ul><li>Ingin menjaga history yang linear dan bersih</li>
<li>Perlu mengupdate branch fitur dengan perubahan terbaru dari main</li>
<li>Ingin menggabungkan beberapa commit menjadi satu (squash)</li>
</ul>
<h2>Contoh Penggunaan</h2></p>
<pre><code class="language-bash"><h1>Pindah ke branch fitur</h1>
git checkout feature-branch
<h1>Rebase dengan main</h1>
git rebase main
<h1>Interactive rebase untuk squash commits</h1>
git rebase -i HEAD~3
</code></pre>
<h2>Tips Penting</h2>
<blockquote>Jangan pernah rebase commit yang sudah di-push ke remote dan digunakan orang lain!</blockquote>
<p>Rebase akan menulis ulang commit history, jadi hanya gunakan pada commit lokal atau branch pribadi.</p>
<h2>Kesimpulan</h2>
<p>Git rebase adalah tool yang powerful untuk menjaga commit history tetap bersih dan mudah dibaca. Dengan workflow yang tepat, tim bisa memiliki history yang informatif dan terstruktur.
