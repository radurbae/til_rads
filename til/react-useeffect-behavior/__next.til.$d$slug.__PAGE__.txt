<h2>Perubahan di React 18</h2></p>
<p>Di React 18 dengan Strict Mode, <code>useEffect</code> akan dipanggil dua kali saat development. Ini untuk membantu menemukan bug yang berkaitan dengan cleanup effect.</p>
<h2>Anatomy of useEffect</h2>
<pre><code class="language-javascript">useEffect(() => {
  // Setup code
  const subscription = dataSource.subscribe();
  
  return () => {
    // Cleanup code
    subscription.unsubscribe();
  };
}, [dataSource]);
</code></pre>
<h2>Best Practices</h2>
<ul><li>Selalu sertakan cleanup function jika setup membuat subscription</li>
<li>Gunakan dependency array dengan benar</li>
<li>Hindari object/array baru di dependency (gunakan useMemo)</li>
</ul>
<h2>Kapan Tidak Perlu useEffect</h2>
<p>Beberapa kasus di mana kamu <strong>tidak</strong> perlu useEffect:
<ul><li>Transformasi data yang bisa dilakukan saat render</li>
<li>Event handlers</li>
<li>Inisialisasi state dari props</li>
</ul>
